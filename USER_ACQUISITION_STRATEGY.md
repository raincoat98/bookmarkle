# 📄 북마클 초기 사용자 확보 전략 & 유료화 정책 제안서

## 📌 개요

북마클(Bookmarkle)은 북마크 관리와 개인화된 묵상/기록 기능을 결합한 생산성 중심 서비스입니다.
현재는 MVP 단계로 기본 기능 검증 및 사용자 확보가 우선이며,
유료 모델은 충분한 사용자 기반 확보 후 단계적으로 적용하는 전략이 가장 효과적입니다.

본 문서는 최근 SaaS/생산성 서비스들의 유료화 트렌드를 바탕으로
북마클에 최적화된 초기 전략 → 유료화 전환 계획까지 포함한 실행안입니다.

---

## 1. 📈 초기 단계: 모든 기능 무료 + 얼리버드 혜택 제공

### 🎯 목표

- 최대한 빠르게 사용자 유입
- 핵심 기능 개선을 위한 피드백 확보
- 정식 버전을 위한 브랜드 신뢰도 확보

### 💡 전략

초기에는 "모든 기능 무료 + 얼리어답터 혜택"이 가장 효과적이며,
아래와 같은 문구를 공식 배너/온보딩에 사용합니다:

#### 🔹 추천 문구 (오피셜)

> **현재 베타 기간으로 모든 기능 무료 제공 중입니다.**
>
> 정식 오픈 이후 일부 기능은 프리미엄으로 전환될 수 있으나,
> 지금 가입한 얼리유저는 기존 기능을 계속 무료로 사용할 수 있습니다.

### 👍 장점

- 사용자 확보 속도 증가
- 향후 유료화 시 저항 감소
- 초기 유저에게 VIP 경험 제공 → 충성 사용자 확보

---

## 2. 🚦 유료화 시점: 사용자 1,000~3,000명 확보 후 진행 추천

### 📌 실제 데이터 기반 기준

- **1,000명 이하** → 유료화 하면 성장 정체
- **1,000~3,000명** → 자연스럽게 유료화 가능
- **5,000명 이상** → 구독 전환율 2~5% 안정적

북마클은 Chrome Extension 기반이라 전환율이 모바일 앱보다 높기 때문에
**1,500~2,000명 선에서 유료 진입**하는 것이 최적입니다.

---

## 3. 🧩 유료 기능 도입 방식: Soft Lock(소프트 락) 전략

유료 기능을 한 번에 모두 잠그는 것이 아니라,
초기에는 무료로 제공하되 정식 오픈 이후 "점진적으로 제한"하는 방식입니다.

### 🔧 Soft Lock 적용 방식 예시

정식 오픈 시 아래 요소들이 제한됩니다:

- **북마크 개수**: 무제한 → 500개 무료 제한
- **컬렉션 개수**: 무제한 → 10개 제한
- **백업 주기**: 매일 → 주 1회로 제한
- **고급 검색 기능** → 프리미엄 이동
- **북마크 통계** → 프리미엄 이동
- **프리미엄 테마/위젯 추가**

### ⭐ 기존 베타 유저 혜택

정식 오픈 이전 가입 사용자에게는 아래 혜택 부여:

> **기존 베타 유저 → 기존 기능 영구 무료 유지 (Free VIP)**

→ 이 방식은 실제로 유료화 반발이 거의 없고, 만족도가 매우 높습니다.

## 4. 🌍 다른 서비스들의 유료화 트렌드

| 서비스   | 초기 전략                | 유료화 방식                   |
| -------- | ------------------------ | ----------------------------- |
| Notion   | 모든 기능 무료           | 트래픽 확보 후 팀 플랜 유료화 |
| Raindrop | 무료 + 얼리버드 Lifetime | 1~2년 후 Premium 모델 도입    |
| Anybox   | 대부분 무료              | 일부 고급 기능만 유료         |
| TickTick | 무료로 성장              | 이후 구독 추가                |
| Pocket   | 무료 사용 중심           | 추천/하이라이트 기능 유료     |

### 📌 공통점

→ 초기에는 "돈 버는 것"보다 "사용자 확보"를 우선순위에 둠
→ 충분한 유입 후 일부 기능을 프리미엄으로 전환

## 5. 🚀 북마클에 최적화된 실행 로드맵

### 🔹 Phase 1 — 베타 기간 (현재)

- 모든 기능 무료
- 얼리유저 영구 무료 혜택 제공
- 사용자 피드백 적극 수집
- 온보딩 최적화
- 크롬 스토어 리뷰/평점 확보

### 🔹 Phase 2 — 정식 오픈 1차 (1,500~2,000명 도달 시)

- 무료 플랜 출시
- 북마크/컬렉션/백업 제한 도입
- 일부 고급 기능 Soft Lock 적용
- 프리미엄 플랜 오픈

### 🔹 Phase 3 — 정식 오픈 2차 (확장 단계)

- 프리미엄 기능 확장 (통계, AI 태그, 팀 공유 등)
- 연간 요금제 도입
- 가격 소폭 인상
  (초기 유저는 기존 가격 유지 - Legacy Price)

## 🔚 결론

현재 북마클은 "초기 사용자 유입"이 가장 중요하므로
지금은 유료화하면 안 되고,
모든 기능을 무료로 제공하면서 얼리유저 혜택을 주는 방식이 최적입니다.

**유입 → 신뢰 확보 → Soft Lock → 구독 전환**

이것이 북마클에 가장 안정적이고 성장 가능한 수익화 전략입니다.
